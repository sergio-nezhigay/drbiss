## Project Overview

This is a custom Shopify theme for `dr-biss.myshopify.com`, built on the **Shapes theme** (v4.3.1) by Switch Themes. It uses Liquid (Shopify's templating language), pre-bundled CSS/JS assets, and Shopify CLI for development.

## Commands

```bash
npm run dev    # Start local dev server (hot reload via Shopify CLI)
npm run push   # Push local changes to the live Shopify theme
npm run pull   # Pull latest theme from Shopify store
```

 Store: `dr-biss.myshopify.com`

## Architecture

### Directory Roles

- **`layout/`** — Top-level HTML shells. `theme.liquid` wraps every page; `password.liquid` and `gift_card.liquid` are standalone layouts.
- **`templates/`** — JSON files that define which sections appear on each page type (homepage, product, collection, article, etc.). Templates reference sections by name.
- **`sections/`** — 80+ self-contained Liquid files, each with an embedded `{% schema %}` block that exposes settings to the Shopify theme editor. These are the main building blocks of pages.
- **`snippets/`** — 130+ small, reusable Liquid partials. Prefixes indicate type: `block-*`, `icon-*`, `product-*`, `media-*`.
- **`assets/`** — Pre-bundled CSS (`base.bundle.css`, `base.bundle.custom.css`) and JS (`global.bundle.js`, `island-*.bundle.js`). **These are build artifacts — do not hand-edit bundles.** Custom CSS overrides go in `base.bundle.custom.css`.
- **`config/`** — `settings_schema.json` defines the theme editor UI; `settings_data.json` stores live values (auto-generated by Shopify admin, treat as read-only).
- **`locales/`** — Translation strings for 10 languages. `en.default.json` is the source of truth; `*.schema.json` files hold editor-facing strings.

### Key Patterns

- **Island architecture:** Interactive JS components are isolated in `island-*.bundle.js` files, keeping the main bundle lean.
- **Color schemes:** Defined in `settings_schema.json` and applied via CSS variables. Sections reference color schemes by index.
- **Section schemas:** Every section's `{% schema %}` block drives its Shopify editor settings. When adding new section settings, add them here and reference them via `section.settings.*` in the Liquid.
- **Snippet includes:** Use `{% render 'snippet-name', param: value %}` — snippets cannot access parent scope directly, so all needed data must be passed explicitly.
- **Theme events:** Custom JS events for cart, drawer, and UI state changes are defined in `assets/theme-events.js`.

## Brand Colors

Defined as CSS variables in `assets/base.bundle.custom.css` on `:root`:

```css
--announcement-bar-bg: #3d6b2e;    /* dark green */
--announcement-bar-color: #f5e09a; /* light gold */
```

Reuse these variables anywhere on the site rather than hardcoding the hex values.

## CSS / Tailwind Note

Tailwind utility classes exist throughout the codebase but there is **no Tailwind compiler running**. The existing classes work because they are already in the pre-built `base.bundle.css`. For any new or modified styles, **use standard CSS** (in `base.bundle.custom.css` or an inline `<style>` block in a section) — do not add new Tailwind utility classes as they will not be compiled.

### Customization Points

- Visual overrides → `assets/base.bundle.custom.css`
- New sections → `sections/` (with schema block)
- New reusable partials → `snippets/`
- Page composition → edit the relevant `templates/*.json`
- Translation strings → `locales/en.default.json` (and mirror to other locales as needed)
